1. Combinación de colores

  • Tus compañeros tienen dudas sobre la combinación de color más accesible entre las siguientes opciones:
    • Color de texto: #1F1FFF y color de fondo: #A3A3A3
    • Color de texto: #000047 y color de fondo: #B8B8B8
Solución:
  • La segunda opción (#000047 sobre #B8B8B8) es la más accesible, ya que cumple con el criterio AA para Texto Normal y AAA para Texto Grande, lo cual es el mínimo recomendado (AA).

3. Deshabilitar imágenes y JavaScript

  • ¿Para qué sirve deshabilitar las imágenes y el JavaScript en una web?
Solución:
  • Se puede ocultar fácilmente las imágenes de las páginas web mediante la extensión Ocultar imágenes disponible en chrome. Esta extensión desplegará un icono junto a la barra de navegación que al pulsarlo oculta automáticamente todas las imágenes de un sitio web. Para deshabilitar el Javascript de una página web, hay que ir a Herramientas de desarrolladores de Chrome. En el menú de comandos escribimos javascript hasta que aparezca la opción deshabilitar javascript. Pulsaremos Enter para ejecutar y se deshabilitará el javascript. También se puede hacer accediendo a configuración en Chrome, Privacidad y seguridad, Configuración de sitios y haciendo click en javascript. Se puede pulsar sobre la opción No permitir que los sitios usen javascript para desactivarlo de todas las páginas web

5. Texto alternativo en las imágenes

  • ¿Le pondrías un texto alternativo al logotipo de una empresa? ¿Qué texto alternativo? ¿y a una imagen decorativa?
Solución:
  • A un logotipo siempre es conveniente acompañarlo de un texto alternativo breve o eslogan descriptivo que acompañe a la imágen. Esto no tan solo es positivo para la accesibilidad, sino como estrategia de comunicación que hace que los clientes recuerden con mayor facilidad la marca. Este texto alternativo le puede indicar al usuario con dificultades para visualizar, que se encuentra en una web de dicha marca. Aunque cabe recalcar que no cualquier texto es válido, solo un texto corto de un par de palabras que encajen con lo que el logo quiere representar, con el nombre de la empresa o la marca en cuestión. En cuanto a una imágen decorativa, lo más aconsejable es no incluirlas para hacer nuestra web más ligera y accesible, pero en caso de introducirla, lo mejor es no acompañarla de texto para no sobrecargar con información innecesaria.

6. Nivel de conformidad de los proyectos

  • Estos son los distintos niveles de conformidad sobre accesibilidad disponibles: ¿Qué nivel de conformidad exigirías para un sitio web muy complejo?
Solución:
  • Existen tres niveles de conformidad: A (Básico), AA (Intermedio) y AAA (Avanzado).
  • El nivel que se debe cumplir es el AA, ya que es el estándar más común y el mínimo legal en muchos países.

7. Separación del estilo y la estructura

  • ¿Por qué es importante separar el estilo (CSS) de la estructura (HTML)?
Solución:
  • Es importante separar los estilos del HTML principalmente, por limpieza , ya que si incluimos los estilos dentro de los elementos HTML, el lector puede verse confundido ante tantos datos juntos. Además, es importante también por mantenimiento, ya que unos estilos separados se hacen más fáciles de actualizar y modificar sin tener que alterar el HTML. Otra ventaja de separar los estilos, es la posibilidad de poder preparar hojas de estilo diferentes, haciendo la página más personalizable. También es importante considerar la posibilidad de hacer una web responsive, que desde los estilos del HTML no se puede lograr.

8. Anidamiento de los encabezados

  • Te han encargado que mejores la accesibilidad de un sitio web que tiene graves problemas de estructuración en su contenido. Esta web no utiliza elementos semánticos ni tiene los encabezados anidados de forma correcta: h1, h2, h3. En su lugar dispone de encabezados desestructurados: h1, h3, h5 e incluso encabezados creados con divs en lugar de usar el elemento semántico correspondiente. ¿Cómo podrías mejorar este diseño? ¿Qué herramienta utilizarías para comprobar el anidamiento de los encabezados?
Solución:
  • Para mejorar el diseño de esa página web, reduciría los h1 a uno por página y emplearía correctamente los encabezados por orden jerárquico de apartados. Sustituiría los divs por sus títulos o etiquetas correspondientes para favorecer su identificación, y me encargaría de la correcta maquetación de la página web. Es decir, distribuiría la página entre sus componentes estándar: header, nav, aside , main, footer, section, article, etc, que hacen una buena estructuración y ayuda a reconocer mejor sus partes. Una buena herramienta para comprobar el correcto anidamiento de una web y su accesibilidad, es la extensión para Chrome AxeDevTools, la cual hace un escaneo de la estructura web ayudando a su correcta estructuración.

11. Atributo accesskey

  • ¿Para qué usarías el atributo acceskey?
Solución:
  • Accesskey se emplea para asignar en HTML un atajo de teclado con Alt a una acción de HTML. Por ejemplo, aplicar el atajo alt+s a un botón (s de send) para enviar un formulario. Es tan fácil de emplear como escribir accesskey=”s” dentro de la etiqueta HTML del elemento. Esta es una herramienta muy poderosa para la accesibilidad que se puede emplear facilitando las tareas a alguien con dificultades de navegación. Se podría emplear en enlaces por ejemplo, inputs, y demás elementos para acudir directamente al elemento o acción referenciados

12. Atributo tabindex

  • ¿Para qué usarías el atributo tabindex? ¿Qué valor puedes darle a tabindex para eliminar un el elemento del orden de pestañas?
Solución:
  • El atributo tabindex se emplea para establecer en el html, el orden de tabulación. Es útil para ayudar al usuario a navegar por la web sin necesidad de usar el ratón, solo tabulando sobre la misma. Es importante establecer mediante este atributo un correcto orden de tabulación para impedir que el usuario se pierda en la navegación. Asignar un orden se hace mediante un número entero dentro de la etiqueta. por ejemplo: < input tabindex=2 > Si en algún momento queremos excluir un elemento del orden de tabulación, tendremos que asignar un valor negativo, por ejemplo: < input tabindex=-1 >

13. Controles del formulario

  • ¿Cómo etiquetarías correctamente los controles del siguiente formulario?
  • <label>Nombre:</label>
    <input type="text" name="textfield" />
  • ¿Cómo haríamos accesibles los inputs que no tienen un label asociado?
Solución:
  • <label for=”textfield” id=”textfield-label”>Nombre:</label> <input type="text" name="textfield" id=”textfield” aria-labelledby=”textfield-label” />
  • En caso de que un input no tenga un <label> visible asociado, se debe usar el atributo `aria-label`, que permite establecer una etiqueta invisible que detectará el lector de pantalla:
    <input type="text" name="textfield" id=”textfield” aria-label=”campo de texto” />
